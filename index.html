Пример 1: "number" + 3 + 3
Result is number33
Цей результат ми отримали через те що перше значення у нас є стрінгом тобто строкою і стоїть на початку, далі усі інші значення додаються як строки церез бінарний плюс, конкатенація

Пример 2: null + 3
Result is 3
null це пустота, порожнина, приводиться до 0 тому при операції залишиться тільки 3

Пример 3: 5 && "qwerty"
Result is qwerty
У цій операції обчислення йде зліва на право, кожен операнд стає булевий, пошук йде поки не знайдеться false, результат це останій правдивий результат.

Пример 4: +'40' + +'2' + "hillel";
Result is 42hillel
У цьому прикладі унарний плюс перетворює рядки на числа (особливо на початку), тому перші два значення додаються, третє не містить числа тому не змінюється а додаєтсья до результата.

Пример 5: '10' - 5 === 6;
Result is False
При операції стрінга стане числом а результат 10-5 не може строго дорівнювати 6

Пример 6: true + false
Result is 1
При арифметичній операції JS приводить все до значень де тру це 1 а фолс це 0

Пример 7: '4px' - 3
Result is NaN
Неможливо від строки відняти якесь число. Так як назва строки містить літери у число вона не переводиться. Тому такий результат.

Пример 8: '4' - 3
Result is 1
При арифметичній операціїї JS все приводить до значення строка стає 4 тому такий результат

Пример 9: '6' + 3 ** 0;
Result is 61
Відбувається конкатенація перше це строка, а друге число у степені, будь яке число у 0 степені це 1 тому такий результат.

Пример 10: 12 / '6' Result is 2
Щоб виконати цю операцію стрінга перетвориться на число тому і такий результат вийде

Пример 11: '10' + (5 === 6);
Result is 10false
У данному прикладі йде по'єднання стрінгів, оскільки перше значення стрінга а далі йде плюс, значення у скобках неправдиве тобто False, яке дорівнює нулю. Але так як йде об'єднання стрінгів отримаємо такий результат

Пример 12: null == ''
Result is False
Нул це завжди хибне значення, стрінга це тру тому цей вираз має такий результат фолс не дорівнює тру

Пример 13: 3 ** (9 / 3);
Result is 27
Ділення у дужках дає результат 3 потім зведення у степінь тому такий результат

Пример 14: !!'false' == !!'true'
Result is True
Подвійне НЕ перевело обидва значення у булевий тип тому вони рівні.

Пример 15: 0 || '0' && 1
Result is 1
Оператор && приоритетніший за "або", до першого хибного тому 1 (таких не було), потім "або" поертає перше правдиве теж 1.

Пример 16: (+null == false) < 1;
Result is False
Плюс приводить нул до 0, фолс також нуль, результат тру, тру це одиниця а одиниця не більша і не меньша за одиницю.

Пример 17: false && true || true
Result is True
Перше наше "І" повертає фолс бо це хибне значення, далі "АБО" повертає тру оскільки це правдиве значення.

Пример 18: false && (false || true);
Result is False
Спочатку у дужках за допомогою "АБО" знаходимо перше правдиве значення, далі "І" повертає фолс бо це перше хибне.

Пример 19: (+null == false) < 1 ** 5;
Result is False
Плюс приводить нул до 0 фолс також нуль, результат тру, тру це одиниця а одиниця не більша за одиницю навіть якщо одиниця у 5-то ступені.
